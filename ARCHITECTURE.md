## 2. 核心模块设计

### 2.1 工作流引擎 (ai-engine)

**职责**: 执行工作流定义的节点图

**核心组件**:

```
src/
├── core/
│   ├── engine.ts        # 工作流执行引擎入口
│   ├── graph-builder.ts # 构建执行图
│   ├── context.ts       # 执行上下文
│   └── variable-resolver.ts # 变量解析
├── nodes/
│   ├── registry.ts      # 节点注册中心
│   ├── base-executor.ts # 节点执行器基类
│   └── executors/      # 具体节点执行器
│       ├── start-executor.ts
│       ├── llm-executor.ts
│       ├── http-executor.ts
│       ├── condition-executor.ts
│       ├── knowledge-executor.ts
│       └── end-executor.ts
├── validators/
│   ├── workflow-validator.ts  # 工作流验证
│   └── node-validators/      # 节点验证
└── knowledge/
    ├── store/           # 向量存储
    ├── retriever/      # 检索器
    ├── chunking/       # 文档切分
    └── embeddings/     # 嵌入模型
```

**执行流程**:

```
1. 加载工作流定义 (nodes + edges)
2. 构建执行图 (GraphBuilder)
3. 从Start节点开始执行
4. 根据节点类型调用对应Executor
5. Executor处理完成后选择下一个节点
6. 重复直到End节点或出错
7. 收集执行结果和trace信息
```

### 2.2 节点执行器设计

每个节点实现 `BaseNodeExecutor`:

```typescript
interface INodeExecutor {
    // 节点类型标识
    nodeType: string

    // 执行节点逻辑
    execute(node: WorkflowNode, context: ExecutionContext): Promise<ExecutionResult>

    // 验证节点配置
    validate(node: WorkflowNode): ValidationResult
}
```

### 2.4 API服务 (api-server)

**职责**: 提供外部API调用接口

**端点设计**:

```
POST /api/workflow/run
  - 认证: API Key (Header: X-API-Key)
  - 参数: inputs (JSON)
  - 返回: 执行结果 + executionId
```

```

    │                         │                       │
```

## 4. 数据库架构

### 4.1 核心表关系

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│    User     │       │    App      │       │  Published  │
│             │◄──────│   (1:N)     │◄──────│    App      │
└─────────────┘       └──────┬──────┘       └──────┬──────┘
                            │                      │
                     ┌──────┴──────┐       ┌──────┴──────┐
                     │   Workflow  │       │ AppExecution│
                     │   (1:N)     │       │   (1:N)     │
                     └─────────────┘       └─────────────┘

┌─────────────┐       ┌─────────────┐
│    App      │       │   ApiKey   │
│   (1:N)     │◄──────│             │
└─────────────┘       └─────────────┘

┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│    User     │       │KnowledgeBase│       │  Document  │
│             │◄──────│   (1:N)     │◄──────│   (1:N)    │
└─────────────┘       └─────────────┘       └─────────────┘
```

### 4.2 数据分区策略

- **用户数据**: 按 userId 隔离
- **应用数据**: App → Workflow → PublishedApp 层级管理
- **执行记录**: WorkflowExecution (测试) / AppExecution (API调用)

## 5. 安全设计

### 5.1 认证鉴权

| 场景     | 认证方式  |
| -------- | --------- |
| 用户登录 | JWT Token |
| API调用  | API Key   |

### 5.2 API Key 设计

```
生成规则: sk-{随机32位字符}
存储: 加密存储实际Key值
显示: sk-{前缀4位}...{后缀4位}
```

### 5.3 数据隔离

- 应用按用户隔离
- API Key 绑定到具体应用
- 外部只能通过API Key调用已发布版本

## 7. 扩展性设计

### 7.1 节点扩展

新增节点只需:

1. 在 `ai-engine` 中创建 Executor 类
2. 注册到 NodeRegistry
3. 在 `workflow` 中创建对应的节点组件和配置表单

### 7.2 模型扩展

通过 LangChain 支持多种 LLM:

- OpenAI
- Anthropic
- Ollama (本地)
- 其他 LangChain 支持的模型
