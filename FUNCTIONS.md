## 2. 功能详述

### 2.1 工作流编辑器

#### 2.1.1 画布编辑

- **节点拖拽**: 从节点面板拖拽节点到画布
- **节点连接**: 从节点输出handle拖拽到目标节点输入handle
- **节点选择**: 点击选中节点，高亮显示
- **多选**: 支持框选多个节点
- **节点删除**: 选中节点按Delete删除
- **画布操作**: 支持缩放、平移、网格吸附
- **快捷键**: Ctrl+S保存、Ctrl+Z撤销等

#### 2.1.2 节点类型

**Start 节点**

- 功能: 定义工作流输入参数
- 配置项:
    - 输入参数名称和类型
    - 默认值（可选）

**LLM 节点**

- 功能: 调用大语言模型生成内容
- 配置项:
    - 模型选择（支持 Ollama 等）
    - System Prompt
    - User Prompt（支持变量引用）
    - Temperature
    - Max Tokens
    - Stream 开关
- 输出: 模型生成文本

**HTTP 节点**

- 功能: 发起 HTTP 请求
- 配置项:
    - 请求方法 (GET/POST/PUT/DELETE)
    - URL（支持变量）
    - Headers
    - Body（支持变量）
    - 超时时间
- 输出: 响应体 JSON

**Condition 节点**

- 功能: 条件分支
- 配置项:
    - 条件表达式（变量 + 比较运算符）
    - 分支名称（true/false 或自定义）
- 输出: 布尔值或选择下一节点

**Knowledge 节点**

- 功能: 知识库检索
- 配置项:
    - 知识库选择
    - 查询内容（支持变量）
    - Top K
    - 阈值
- 输出: 检索到的文档片段

**End 节点**

- 功能: 定义工作流输出
- 配置项:
    - 输出字段映射
    - 输出格式

#### 2.1.3 节点配置面板

- 动态表单渲染，根据节点类型显示不同配置项
- 变量智能提示（输入 @ 或 {{ 触发）
- 实时验证配置有效性
- 自动保存配置

#### 2.1.4 测试运行

- **输入配置**: 填写 Start 节点输入参数
- **执行控制**: 开始/停止执行
- **结果展示**: 实时显示各节点执行结果
- **Trace追踪**: 显示节点执行顺序和耗时
- **执行历史**: 保存历史执行记录，可回溯查看

### 2.2 知识库模块

#### 2.2.1 知识库创建

- 知识库名称、描述、图标
- 嵌入模型配置:
    - 模型选择（默认: mxbai-embed-large）
    - 向量维度（默认: 1024）
- 切分配置:
    - Chunk Size（默认: 500）
    - Chunk Overlap（默认: 50）

#### 2.2.2 文档管理

- **上传文档**: 支持 PDF、TXT、Markdown 等格式
- **文档列表**: 显示已上传文档及状态
- **文档处理**: 自动提取文本、切分、向量化
- **删除文档**: 删除指定文档

#### 2.2.3 检索配置

- 检索模式: 向量检索 / 全文检索 / 混合检索
- 向量权重: 0-1 浮点数
- Top K: 召回数量（默认: 5）
- 相似度阈值: 0-1 浮点数

#### 2.2.4 检索测试

- 输入查询文本
- 实时返回检索结果
- 显示相关性分数

---

### 2.3 应用管理

#### 2.3.1 应用CRUD

- **创建应用**: 选择类型、填写名称、描述、图标
- **应用类型**:
    - WORKFLOW（工作流）
    - CHATBOT（聊天机器人）
    - AGENT（智能体）
- **编辑应用**: 修改应用基本信息
- **删除应用**: 软删除，支持恢复

#### 2.3.2 应用发布

- 发布当前工作流为新版本
- 版本号自动递增
- 发布后可被 API 调用
- 支持多版本管理

#### 2.3.3 API密钥管理

- **创建密钥**: 生成新的 API Key
- **密钥列表**: 显示所有密钥
- **密钥详情**:
    - 密钥名称
    - 密钥前缀（sk-xxxx...）
    - 启用/禁用状态
    - 过期时间
    - 使用次数
    - 最后使用时间
- **禁用/启用**: 可随时开关密钥
- **删除密钥**: 删除指定的密钥

### 2.4 API服务

#### 2.4.1 调用接口

```
POST /api/workflow/run
Header: X-API-Key: sk-xxxxx
Body: {
  "inputs": {
    "参数1": "值1",
    "参数2": "值2"
  }
}
```

#### 2.4.2 响应格式

```json
{
    "success": true,
    "executionId": "exec_xxx",
    "outputs": {
        "result": "输出内容"
    },
    "duration": 1234,
    "totalTokens": 100
}
```

#### 2.4.3 执行追踪

- 记录每次 API 调用的输入输出
- 保存节点执行详情（nodeTraces）
- 记录执行耗时和 Token 消耗

## 6. 用户交互

### 6.1 工作流编辑器交互

```
1. 打开应用 → 进入工作流编辑器
2. 从左侧面板拖拽节点到画布
3. 点击节点 → 右侧弹出配置面板
4. 配置节点参数（支持变量引用）
5. 从输出handle拖拽到输入handle连接节点
6. 点击顶部「测试」按钮
7. 填写输入参数
8. 点击「运行」查看结果
9. 调试完成后点击「发布」
10. 在API密钥管理创建密钥
11. 通过API调用已发布应用
```

### 6.2 知识库交互

```
1. 进入应用 → 点击「知识库」
2. 点击「新建知识库」
3. 填写名称和配置项 → 创建
4. 点击「上传文档」上传文件
5. 等待处理完成（自动提取、向量化）
6. 点击「测试检索」验证效果
7. 在工作流中使用Knowledge节点
```
